Login Page Arabic Translation Test

Test: Arabic Label Translation
This test verifies that the email and password labels change to Arabic when the language is switched.
Step 1: Open the application at http://localhost:5175
Step 2: You should see the login page with English labels
Step 3: Click the "العربية" button in the top-right corner
Step 4: Observe the form labels
Expected Result:
- When English is selected: "Email Address" and "Password" labels
- When Arabic is selected: "عنوان البريد الإلكتروني" and "كلمة المرور" labels
- The login button should also change from "Sign In" to "تسجيل الدخول"
- The form should be right-aligned when Arabic is selected
Translation Keys Being Tested
The following translation keys should work correctly:
email_label: "Email Address" → "عنوان البريد الإلكتروني"
password_label: "Password" → "كلمة المرور"
sign_in: "Sign In" → "تسجيل الدخول"
email_placeholder: "Enter your email address" → "أدخل عنوان بريدك الإلكتروني"
password_placeholder: "Enter your password" → "أدخل كلمة المرور"
Recent Changes Made
The following changes were made to fix the translation issue:
1. Added cache clearing when language changes in TranslationProvider
2. Added version tracking to force re-renders when language changes
3. Added language change event listener in Login component
4. Added form key with currentLanguage and forceUpdate to ensure re-rendering
