üìã Problem Analysis

The authorization error occurs because:

  • The user needs the create_staff permission
  • Permissions might not be seeded in the database
  • The user's role might not have the required permissions

üî® Solution Steps

Step 1: Reseed Permissions

Run this command to ensure all permissions are created and assigned to roles:

cd backend/wesaltech
php artisan db:seed --class=RolePermissionSeeder

Step 2: Check Your User's Role

Verify your user has the correct role in the database:

php artisan tinker

# Check your user's roles and permissions
$user = User::where('email', 'your-email@example.com')->first();
$user->roles->pluck('name');
$user->getAllPermissions()->pluck('name');

Step 3: Assign Role if Missing

If your user doesn't have a role, assign one:

php artisan tinker

$user = User::where('email', 'your-email@example.com')->first();
$user->assignRole('tenant_admin');

# Verify
$user->hasPermissionTo('create_staff'); // Should return true

Step 4: Clear Cache

Clear the permission cache:

php artisan permission:cache-reset
php artisan cache:clear

üéØ Quick Fix Command

cd backend/wesaltech && php artisan db:seed --class=RolePermissionSeeder && php artisan permission:cache-reset && php artisan cache:clear

üí° Additional Debugging

If the issue persists, check:

  • Your user's tenant_id is set correctly
  • The model_has_roles table has entries for your user
  • The role_has_permissions table has the correct permissions

SQL Query to check:

SELECT u.email, r.name as role, p.name as permission
FROM users u
JOIN model_has_roles mhr ON u.id = mhr.model_id
JOIN roles r ON mhr.role_id = r.id
JOIN role_has_permissions rhp ON r.id = rhp.role_id
JOIN permissions p ON rhp.permission_id = p.id
WHERE u.email = 'your-email@example.com'
AND p.name LIKE '%staff%';
